BlackRoad API

Complete API reference for BlackRoad OS platform

Introduction

The BlackRoad API provides programmatic access to our AI, quantum computing, and infrastructure platforms. All endpoints use REST principles and return JSON responses.

Base URL

https://api.blackroad.io/v1

Authentication

All API requests require authentication using an API key. Include your key in the Authorization header.

Example Request

curl https://api.blackroad.io/v1/ai/models \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
POST /ai/inference

Run inference on an AI model with custom input data.

Parameters

Parameter Type Required Description
model string Yes Model ID to use for inference
input object Yes Input data for the model
parameters object No Model-specific parameters

Request Example

JSON

{
  "model": "lucidia-qi-v1",
  "input": {
    "text": "Explain quantum entanglement"
  },
  "parameters": {
    "temperature": 0.7,
    "max_tokens": 500
  }
}

Response

200 Success

JSON

{
  "id": "inf_abc123",
  "model": "lucidia-qi-v1",
  "output": {
    "text": "Quantum entanglement is a phenomenon where..."
  },
  "usage": {
    "tokens": 342
  },
  "created_at": "2025-12-26T10:30:00Z"
}

POST /quantum/circuit

Create and execute a quantum circuit.

Request Example

JSON

{
  "qubits": 3,
  "gates": [
    { "type": "H", "target": 0 },
    { "type": "CNOT", "control": 0, "target": 1 },
    { "type": "CNOT", "control": 1, "target": 2 }
  ],
  "shots": 1000
}

Response

200 Success

JSON

{
  "circuit_id": "qc_xyz789",
  "results": {
    "000": 503,
    "111": 497
  },
  "execution_time": "0.234s",
  "status": "completed"
}
GET /data/metrics

Retrieve system metrics and analytics.

Query Parameters

Parameter Type Description
from timestamp Start time (ISO 8601)
to timestamp End time (ISO 8601)
interval string Aggregation interval (1m, 5m, 1h, 1d)

Response

JSON

{
  "metrics": [
    {
      "timestamp": "2025-12-26T10:00:00Z",
      "requests": 1523,
      "latency_p95": "42ms",
      "errors": 3
    }
  ],
  "interval": "1m",
  "from": "2025-12-26T10:00:00Z",
  "to": "2025-12-26T11:00:00Z"
}

Error Handling

The API uses standard HTTP status codes and returns detailed error messages in JSON format.

Error Response Format

JSON

{
  "error": {
    "code": "invalid_request",
    "message": "Missing required parameter: model",
    "details": {
      "parameter": "model",
      "location": "body"
    }
  }
}

Common Error Codes

Status Code Description
400 invalid_request Request validation failed
401 unauthorized Invalid or missing API key
500 internal_error Server error occurred
